================================================================================
SLIDE: TECH-LEVEL ARCHITECTURE
================================================================================

"Before we jump into the live demo, let me quickly walk you through how the
system is actually built — because this is what makes it powerful."

[Point to Layer 1]
"At the top we have the Web Interface — built with Flask on the backend,
streaming live data to the browser using Server-Sent Events, with plain
HTML, CSS, and JavaScript on the frontend. Lightweight, fast, no heavy
frameworks."

[Point to Layer 2]
"Below that is the Ingestion layer — Phases 1 through 4. This is where raw
PCAP files are validated, parsed using capinfos, and all metadata is stored
in a local SQLite evidence registry database."

[Point to Layer 3]
"Layer 3 is Extraction — Phase 5. This is where tshark, Wireshark's
command-line engine, deep-dives into the packet captures and pulls out
structured network events. We parallelize this with Python multiprocessing
across up to 4 workers simultaneously to handle large captures fast."

[Point to Layer 4]
"Layer 4 is where the AI lives — Phases 6 through 10. This is the agentic
reasoning engine. We're using Ollama running Llama 3.1 locally — all
on-device, no cloud, no data leaving the machine. The AI receives structured
evidence in JSON Schema format and reasons about it using the MITRE ATT&CK
framework — the same threat taxonomy used by professional security teams
worldwide."

[Point to Layer 5]
"And finally Layer 5 — Correlation and Reporting. The agent's findings are
correlated across bundles, and the output is rendered as a Mermaid flowchart,
a Markdown report, and structured JSON — ready for analysts or further
automation."

"Five layers, fully automated, fully local. Let's see it in action."

PAGE 2 — PIPELINE  (http://192.168.1.240:5000/pipeline)
================================================================================

"Now we move to the Pipeline Control page — the engine room of the system.
This is where we configure and fire off the analysis."

[Point to Phase Range selectors]
"The pipeline has 11 phases in total. We can choose exactly which range to
run. For a full end-to-end analysis — ingestion through AI reasoning and
report generation — we run Phase 1 through 9. If we've already ingested
the files and just want to re-run the AI layer, we can jump straight to
Phase 6."

[Point to Workers input]
"The Workers setting controls Phase 5 — the tshark extraction step. Setting
this to 4 means we're running 4 parallel extraction threads simultaneously,
matching Layer 3 in the architecture. This is what lets us chew through
119 PCAPs efficiently."

[Toggle Agentic Mode ON]
"Here is the key feature — Agentic Mode. When I flip this toggle, the
pipeline activates the LLM reasoning engine in Phases 6 through 10.
We're running Llama 3.1 locally via Ollama — you can see the model name
right here. No data ever leaves this machine."

"The AI doesn't just match traffic against a fixed set of rules. It actively
hypothesizes — it weighs the evidence, considers alternative explanations,
assigns confidence scores, and reasons about what an attacker might be doing.
That's what makes this agentic rather than just automated."

[Click Start Pipeline]
"I'll hit Start now. Watch the Live Output panel on the right — this is a
real-time log stream using Server-Sent Events. Every single line the pipeline
prints is pushed directly to the browser as it happens, no polling, no
refresh needed."

[Point to status badge]
"The status badge flips from Idle to Running. If we ever need to abort, the
Stop button is right there and will cleanly terminate the process."

"You can watch each phase complete in sequence — ingestion, parsing,
extraction, and then the AI phases where you'll see the LLM generating its
reasoning live in this terminal."

PAGE 4 — RESULTS  (http://192.168.1.240:5000/results)
================================================================================

"Now the most important part — the Investigation Results page. Every PCAP
the pipeline analyzed is grouped into what we call a bundle — a
self-contained investigation package. Our 119 PCAPs produced 52 bundles."

[Point to the bundle cards grid]
"Each card in this grid represents one bundle. At a glance you get:
- The number of confirmed security Findings — shown in red if any were found
- The number of AI-generated Hypotheses
- The total network Events analyzed
- How many PCAPs are in this bundle
- Whether this was an Agentic LLM run or a rule-based run — you can see
  the LLM badge on agentic bundles
- And the analysis runtime at the bottom"

"52 bundles, all generated automatically. Let me open one."

[Click into a bundle]

"Inside a bundle we get the full forensic investigation report."

[Point to the four summary cards at the top]
"Summary at the top: findings count, hypotheses, events analyzed, and
analysis runtime for this specific bundle."

[Scroll to Findings section]
"The Findings section shows confirmed security issues. Each finding has:
- A title describing what was detected
- A severity level — Critical, High, Medium, or Low — color coded
- A confidence score
- MITRE ATT&CK technique tags — for example T1071 for Application Layer
  Protocol abuse, or T1046 for Network Service Scanning

This is the language professional security teams use. Our AI maps directly
to this framework, so findings slot straight into any existing SOC workflow."

[Scroll to Hypotheses section]
"Below findings are the AI's Hypotheses. These are the agent's working
theories — ideas it considered that may or may not have crossed the
confidence threshold to become a confirmed finding. Each one has a
confidence bar and the agent's written reasoning for why it believes
this behaviour is happening."

"This is valuable because a human analyst can look at a high-confidence
hypothesis that didn't quite make the threshold and decide whether to
escalate — the AI doesn't make the final call, it informs it."

[Scroll to LLM Reasoning section]
"Here is the raw LLM Reasoning output. This is the agent's chain of thought —
you can literally read what the AI was considering as it evaluated the
evidence. Full transparency into the agentic decision-making process. No
black box."

[Click Show on the Markdown Report]
"Finally, there's a complete Markdown report — a full human-readable
forensics document ready to hand off to an analyst, paste into a ticket,
or include in an incident report. Everything in one place."

"You can click through the entire timeline to investigate any point in time.

[Scroll to Rankings Grid]

All of this derived automatically by the AI from raw packet captures.
No signatures written, no rules tuned, no analyst time spent."

PRESENTER TIPS
================================================================================

- Have at least one PCAP already uploaded before you start — don't wait
  for uploads during the demo

- If the pipeline takes too long to run live, have a completed run ready
  and jump straight to Results and Master Report

- The Hypothesis Timeline on the Master Report is the most visually
  impressive part — spend the most time here, click multiple circles

- When showing Findings, point out the MITRE tags specifically — audiences
  with security backgrounds will immediately understand the value

- Emphasise "on-device, no cloud" whenever you mention the LLM — this is
  a key differentiator for security-conscious audiences

- The LLM Reasoning section (raw chain of thought) surprises audiences —
  pause and let them read a few lines themselves
